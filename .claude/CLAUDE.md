# Obsidian Vault 運用ガイド

## ディレクトリ構成と運用方針

### 基本方針

- **MOC（Map of Contents）は使用しない**
  - 過去に検討した結果、ディレクトリ管理を複雑にしたくないため不採用

### ディレクトリ構成

#### `#data/`
- **用途**: ナレッジベースの構築
- **運用方法**: Web Clipper を使用して AI が要約した内容を格納
- **目的**: 外部情報を体系的に蓄積する

#### `#log/`
- **用途**: 学習セッションの記録
- **運用方法**: `#data/` の内容を AI に質問してもらい、ユーザーの回答をまとめたドキュメントを保存
- **目的**: インタラクティブな学習プロセスを可視化する

#### `notes/`
- **内容**: 過去にインプットした情報をまとめ直したドキュメント
- **位置づけ**: vault 統合時に既存のメモから移行したもの
- **運用**: 参照用として保持（上記の運用フローとは異なる経緯で作成）

### Claude Code での作業時の注意点

- 新規ドキュメント作成時は、上記の運用フロー（`#data/` または `#log/`）に従うこと
- `#data/` と `#log/` の使い分けを意識すること
- `notes/` にある既存ドキュメントは参照のみで、積極的な編集は推奨しない（運用フローが異なるため）
