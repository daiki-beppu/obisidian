# 双方向リンクのパターン集

## 良いパターン

### 1. 冒頭での文脈説明

**パターン**：
```markdown
このドキュメントは [[リンク先タイトル]] の学習セッションで得た理解をまとめたものです。
```

**実例**：
```markdown
このドキュメントは [[2026-01-03T220846+0900_データ_Claude Code Best Practices|Claude Code Best Practices]] の学習セッションで得た理解をまとめたものです。
```

**複数リンクの場合**：
```markdown
このドキュメントは [[リンク1]] および [[リンク2]] を踏まえた学習セッションの記録です。
```

### 2. 補足での詳細誘導

**パターン**：
```markdown
この〇〇の詳細は [[リンク先]] で解説されています。
```

**実例**：
```markdown
この設定管理の詳細は [[2026-01-03T220846+0900_データ_Claude Code Best Practices|Best Practices]] で解説されています。
```

**バリエーション**：
- 「このコマンドを含む〇〇については [[リンク]] で網羅的に紹介されています」
- 「〇〇の実践例は [[リンク]] でも紹介されています」
- 「〇〇の活用方法は [[リンク]] で実践的に解説されています」

### 3. まとめでの関連性明示

**パターン**：
```markdown
[[リンク先]] では、〇〇の重要性が強調されています
```

**実例**：
```markdown
[[2026-01-03T221944+0900_データ_2025年、AI時代の要件定義について考える|AI時代の要件定義]] では、人間の「決断」と「責任」の重要性が強調されています
```

**複数リンク統合の場合**：
```markdown
[[リンク1]] と [[リンク2]] を組み合わせることで、〇〇を整理することができました。
```

### 4. 文中での自然な参照

**パターン**：
```markdown
今回のやり取りを通じて、[[リンク1]] と [[リンク2]] を組み合わせることで...
```

**実例**：
```markdown
今回のやり取りを通じて、[[2026-01-03T221944+0900_データ_2025年、AI時代の要件定義について考える|要件定義の本質]] と [[2026-01-03T220846+0900_データ_Claude Code Best Practices|Claude Code の実践的な活用法]] を組み合わせることで...
```

## 避けるべきパターン

### 1. リンク単体のみ

**ダメな例**：
```markdown
[[リンク先]]
```

**理由**：なぜ関連するのか、どういう文脈でリンクしているのかが分からない

### 2. 箇条書きリスト形式のみ

**ダメな例**：
```markdown
関連ドキュメント：
- [[リンク1]]
- [[リンク2]]
- [[リンク3]]
```

**理由**：各リンクとの関連性が説明されていない

### 3. 文脈なしの参照

**ダメな例**：
```markdown
詳細は [[リンク先]]
```

**理由**：「詳細」だけでは何の詳細かが不明瞭

**改善例**：
```markdown
この設定管理の詳細は [[リンク先]] で解説されています
```

## ディレクトリ別の運用ルール

### data/
- 参照される前提で作成する
- 他のドキュメントからリンクされることを想定した構造にする

### log/
- 必ず文章中でリンクを使用する
- リンク単体は置かない
- 「なぜ関連するか」が後から読んで分かる形で記述する
