# Obsidian 双方向リンク作成ガイド

## 概要

Obsidian vault での学習ログ（log/ ディレクトリ）作成時に、data/ ディレクトリのナレッジベースへ適切な双方向リンクを作成する。

**基本原則**：
- 学習文脈を残すためにリンクを使用する
- data/ は参照される前提で作成する
- log/ は必ず文章中でリンクし、リンク単体は置かない
- 「なぜ関連するか」が後から読んで分かる形で記述する

## 基本ルール

### 1. リンクは常に文脈付きで記述

**良い例**：
```markdown
このドキュメントは [[リンク先]] の学習セッションで得た理解をまとめたものです。
```

**悪い例**：
```markdown
[[リンク先]]
```

### 2. リンク単体のみの行を作らない

リンクは必ず文章の一部として埋め込む。箇条書きリストとしてリンクだけを並べることも避ける。

### 3. 関連性を明示する

各リンクについて、「なぜこのドキュメントとリンクするのか」が読み手に伝わる文脈を提供する。

## リンク作成パターン

詳細なパターンと実例は [link_patterns.md](references/link_patterns.md) を参照。

### 主要パターン

1. **冒頭での文脈説明**：ドキュメントの由来を説明する際にリンク
2. **補足での詳細誘導**：本文中で関連する詳細情報へ誘導
3. **まとめでの関連性明示**：複数のソースを統合する際にリンク
4. **文中での自然な参照**：議論の流れの中で自然に参照

## 実践ガイド

### log/ ファイル作成時

1. **冒頭で元ソースを明示**
   - どの data/ ファイルを基にした学習セッションかを文脈付きでリンク
   - 複数のソースがある場合は「および」「を踏まえた」などで繋ぐ

2. **本文中での補足リンク**
   - 詳細情報がある場合は「〇〇の詳細は [[リンク]] で解説されています」形式
   - 実例がある場合は「〇〇の実践例は [[リンク]] でも紹介されています」形式

3. **まとめセクションでの統合**
   - 複数のソースを組み合わせて理解した場合は、その関連性を明示
   - 各リンクが何を提供しているかを簡潔に説明

### チェックリスト

作成後、以下を確認：
- [ ] すべてのリンクに文脈説明がある
- [ ] リンク単体だけの行がない
- [ ] 「なぜ関連するか」が読み手に伝わる
- [ ] 箇条書きリストとしてリンクだけを並べていない

## ディレクトリ別の役割

### data/
参照される前提で作成されたナレッジベース。Web Clipper や AI 要約を使用して外部情報を蓄積。

### log/
学習セッションの記録。data/ の内容を基に質問と回答をまとめる。双方向リンクで学習文脈を保持。
