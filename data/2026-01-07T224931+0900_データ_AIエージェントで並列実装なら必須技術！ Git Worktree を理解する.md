# AIエージェント時代の並列開発に必須！Git Worktreeの活用

AIエージェント（Claude Code, GitHub Copilotなど）が複数のタスクを並行処理する現代において、従来のGitワークフローが抱える課題を解決し、開発効率を向上させる「Git Worktree」について解説します。

## 従来のGitワークフローの課題
従来のGitワークフローでは、ブランチ切り替え時に以下のような問題がありました。

*   **作業の中断**: 緊急タスク発生時、進行中の作業を一時コミットまたはstashする必要がある。
*   **環境再構築のオーバーヘッド**: ブランチ切り替えごとに依存関係のインストールやビルドが必要な場合がある。
*   **AIエージェントとの相性の悪さ**: AIエージェントが長時間タスクを実行中に、ブランチ切り替えでコンテキストが失われる。

AIエージェント時代では、複数のAIエージェントが同時に長時間タスクを実行し、並列処理による効率化が求められます。

## Git Worktreeとは

Git Worktreeは、**同一リポジトリの複数のブランチを、異なるディレクトリで同時にチェックアウトできる**機能です。

例:

```
/project-main (main branch)
/project-feature-a (feature-a branch)
/project-hotfix (hotfix branch)
```

これにより、別々の作業ディレクトリで独立したブランチの作業を同時に進めることが可能になります。

## Claude Codeでの推奨と使用例

AnthropicのClaude Codeドキュメントでも、並列セッションにGit Worktreeの利用が推奨されています。利点は以下の通りです。

*   **完全な分離**: 各Claude Codeインスタンスが独立したファイル状態を持つ。
*   **干渉の防止**: 複数のセッションが互いに影響しない。
*   **効率的な並列処理**: 長時間タスクを実行しながら他の作業を継続できる。

**使用例**:

```bash
git worktree add ../project-claude-task1 -b claude/task1
git worktree add ../project-claude-task2 -b claude/task2

cd ../project-claude-task1
claude

# 別のターミナルで
cd ../project-claude-task2
claude
```

## おすすめVSCode拡張: Git Worktree Manager

VSCodeでGit Worktreeを直感的に管理できる拡張機能「[Git Worktree Manager](https://marketplace.visualstudio.com/items?itemName=jackiotyu.git-worktree-manager)」が紹介されています。

**主な特徴:**

*   直感的なUIでworktreeを管理。
*   ワンクリックでworktree間の切り替えが可能。
*   各worktreeのGit状態を一覧で確認できる。

**主要機能:**

1.  **Worktreeの作成**
2.  **ブランチ切り替え** (Worktree間の移動)
