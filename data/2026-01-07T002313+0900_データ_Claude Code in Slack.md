# Claude Code in Slack: 概要と活用ガイド

Claude Code in Slackは、Claude Codeの機能をSlackワークスペースにもたらし、チームの会話内で直接コーディングタスクを効率的に実行できます。

## 💡 主な機能
-   **自動検出**: @Claudeメンションでコーディングタスクの意図を自動検出し、Web上のClaude Codeセッションを自動作成します。
-   **コンテキスト収集**: スレッドやチャンネルの会話から関連コンテキストを理解し、タスクに活用します。
-   **進捗状況の共有**: Slackスレッドで作業のステータス更新や完了通知を受け取れます。

## 🚀 主なユースケース
-   バグの調査と修正
-   コードレビューと小規模な機能変更
-   チームでの共同デバッグ
-   バックグラウンドでの並行タスク実行

## ✅ 前提条件
-   **Claudeプラン**: Pro, Max, Team, Enterprise (Claude Codeアクセス権付き)
-   **Claude Code on the web**: アクセスが有効であること
-   **GitHubアカウント**: 少なくとも1つのリポジトリがClaude Codeに接続されていること
-   **Slack認証**: SlackアカウントがClaudeアカウントにリンクされていること

## 🛠️ セットアップ手順
1.  **ClaudeアプリをSlackにインストール**: ワークスペース管理者がSlack App Marketplaceからインストール。
2.  **Claudeアカウントを接続**: Claude App HomeでSlackとClaudeアカウントを連携。
3.  **Claude Code on the webを設定**: `claude.ai/code`でGitHubアカウントとリポジトリを接続。
4.  **ルーティングモードを選択**: App Homeで以下のいずれかを選択。
    *   **Code only**: 全ての@メンションをClaude Codeセッションにルーティング。
    *   **Code + Chat**: メッセージを分析し、コーディングタスクはCode、その他はChatにルーティング。

## 🔄 動作の仕組み
1.  **開始**: @Claudeをメンションしてコーディングリクエストを送信。
2.  **検出**: Claudeがコーディング意図を検出し、Web上のClaude Codeセッションを自動作成。
3.  **更新**: Slackスレッドで作業進捗を通知。
4.  **完了**: 作業完了後、Slackにサマリーとアクションボタン（View Session, Create PRなど）を投稿。

## ⚙️ ユーザーインターフェース要素
-   **App Home**: 接続状況、アカウント接続/切断。
-   **メッセージアクション**: `View Session`, `Create PR`, `Retry as Code`, `Change Repo`。
-   **リポジトリ選択**: コンテキストに基づいて自動選択、または手動で変更可能。

## 🔒 アクセスと権限
-   **ユーザーレベル**: 各ユーザーのClaudeアカウント、プラン制限、GitHubリポジトリアクセスに紐付け。
-   **ワークスペース管理者**: Claudeアプリのインストールを管理。

## 📝 ベストプラクティス
-   **効果的なリクエスト**: ファイル名、関数名、エラーメッセージを含め、具体的に。完了の定義（テスト、PR作成など）を明確に。
-   **Slack vs. Web**: 
    *   **Slack**: 既存のSlack議論がある場合、非同期タスク、チームへの可視性が必要な場合。
    *   **Web**: ファイルアップロード、リアルタイムインタラクション、複雑なタスク。

## トラブルシューティング
-   **セッションが開始しない**: Claudeアカウント、Claude Code on the webアクセス、GitHubリポジトリ接続を確認。
-   **リポジトリが表示されない/選択ミス**: `claude.ai/code`でリポジトリ接続と権限を確認。「Change Repo」ボタンやリクエストにリポジトリ名を含めることで対応。
-   **認証エラー**: Claudeアカウントの再接続、正しいアカウントでのログイン、Claude Codeアクセス権のあるプランかを確認。

## ⚠️ 現在の制限事項
-   **GitHubのみ**: 現時点ではGitHubリポジトリのみをサポート。
-   **単一PR**: 1セッションにつき作成できるプルリクエストは1つ。
-   **レート制限**: 個人のClaudeプランのレート制限が適用されます。
-   **Webアクセス必須**: Claude Code on the webへのアクセスがないユーザーは、通常のClaude Chat応答のみを受け取ります。
