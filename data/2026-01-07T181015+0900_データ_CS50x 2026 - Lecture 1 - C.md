# CS50x 2026 - Lecture 1 - C の要約

この動画は、ハーバード大学が提供するコンピュータサイエンスとプログラミングの入門講座CS50の2026年度版の最初の講義です。C言語プログラミングの基礎と実践的な側面を幅広く紹介しています。

## 主要な学習内容

*   **C言語の導入**: プログラミング言語Cの基本的な概念を学びます。
*   **開発環境とツール**: 
    *   Visual Studio Code for CS50の利用方法
    *   コマンドラインインターフェース（CLI）を介したコードの記述、コンパイル、実行
    *   ターミナルコマンドの基本
*   **プログラミングの基本要素**: 
    *   **Hello, World**: 最初のC言語プログラムの作成
    *   **ScratchからCへ**: グラフィカルなScratchの概念をC言語のコードに変換
    *   **ヘッダーファイルとCS50マニュアルページ**: C言語の標準ライブラリとドキュメントの活用
    *   **Hello, You**: ユーザーからの入力を受け付けるプログラムの作成
    *   **条件分岐 (Conditionals)**: `if`, `else if`, `else` を使った条件判断
    *   **データ型 (Types)**: `int`, `float`, `char`, `bool`, `string` などの基本的なデータ型とその使用
    *   **ループ (Loops)**: `while`, `for` を使った繰り返し処理
    *   **関数 (Functions)**: コードのモジュール化と再利用
*   **コードの品質と実践的な課題**: 
    *   **正確性、設計、スタイル**: 読みやすく、効率的で、バグの少ないコードを書くための原則
    *   **Mario**: マリオのピラミッドをC言語で作成する課題の紹介
*   **数値計算の落とし穴**: 
    *   **整数オーバーフロー (Integer Overflow)**: 数値が表現できる範囲を超えた際の問題
    *   **切り捨て (Truncation)**: 整数型への変換による数値の損失
    *   **型キャスト (Type Casting)**: データ型を明示的に変換する方法
    *   **浮動小数点数の不正確さ (Floating-Point Imprecision)**: 浮動小数点数計算における精度の限界

CS50 2026 第1週：C言語の基礎
本講義では、第0週で学んだScratchの視覚的な概念を、テキストベースのプログラミング言語であるC言語へと移行させ、コンピュータサイエンスの核心的な概念を学びます,。
1. プログラムの仕組み：ソースコードから実行まで
人間が書くプログラムは「ソースコード」と呼ばれますが、コンピュータは「マシンコード（0と1のバイナリ）」しか理解できません。
• コンパイラ (Compiler): 人間が書いたソースコードを、コンピュータが理解できるマシンコードに変換するための特殊なソフトウェアです。
• CLI (コマンドラインインターフェース): マウスで操作するGUIとは異なり、キーボードでコマンドを入力して操作する画面です。VS Code内のターミナルを使用してプログラムのコンパイルや実行を行います,。
2. C言語の基本文法
C言語で「hello, world」を表示するだけでも、Scratchとは異なり多くの厳密な構文が必要になります。
• 関数の構造: printfなどの関数を使用し、文の最後には**セミコロン (;)**を付ける必要があります,。
• エスケープシーケンス: \n は改行を意味し、出力の見た目を整えるために多用されます,。
• ライブラリとヘッダーファイル: 他人が書いたコードを再利用するために #include <stdio.h> のようなヘッダーファイルを読み込みます。CS50独自のライブラリ（cs50.h）も学習の補助として使用されます,,。
3. 変数・データ型・ユーザー入力
プログラム内でデータを扱うための仕組みについて学びます。
• データ型: 整数を表す int、テキストを表す string、単一の文字を表す char、小数点を含む float や double、より大きな整数を扱う long などがあります,,。
• 入力関数: get_string や get_int を使い、ユーザーから対話的に値を取得します,。
• フォーマット指定子: printf 内で %s（文字列）や %i（整数）をプレースホルダーとして使い、変数の値を埋め込みます,。
4. 制御構造（条件分岐とループ）
プログラムの論理的な流れを制御します。
• 条件分岐 (Conditionals): if、else if、else を使って条件に応じた処理を行います。比較には ==（等しい）や >（より大きい）などの演算子が使われます,,。
• ループ (Loops): while（条件が真の間繰り返す）や for（回数を指定して繰り返す）があります。また、最低1回は実行する do-while ループも紹介されました,,。
5. 抽象化と関数
複雑な問題を小さく分解し、再利用可能な「関数」を自分で作成する方法を学びます。
• スコープ (Scope): 変数が存在する有効範囲のことです。ある関数やループ内で作成された変数は、その外側からはアクセスできません,。
• プロトタイプ (Prototype): 自作関数を main 関数の下に書く場合、プログラムの冒頭でその関数の存在を宣言する必要があります,。
6. コンピュータの物理的な限界
ハードウェアの制限によって生じる問題についても触れられました。
• 整数オーバーフロー (Integer Overflow): 割り当てられたビット数（32ビットなど）を超える大きな値を扱おうとすると、値が正しく保持できなくなる現象です。ボーイング787のバグや西暦2038年問題が例として挙げられました,,。
• 浮動小数点の不精度 (Floating-point Imprecision): 有限のビット数では 1/3 などの無限小数を正確に表現できず、近似値になってしまう現象です,。

重要なLinuxコマンド
ターミナルで使用する基本的なコマンドです：
• ls: 現在のフォルダ内のファイル一覧を表示。
• cd: ディレクトリ（フォルダ）を移動。
• mkdir: 新しいディレクトリを作成。
• mv: ファイルの移動や名前の変更。
• rm: ファイルの削除。
