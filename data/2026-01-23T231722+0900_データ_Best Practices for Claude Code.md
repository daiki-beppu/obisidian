# Claude Code 活用ベストプラクティスガイド

Claude Codeを最大限に活用し、自律的なコーディング環境を効率化するための重要ポイントをまとめます。

### 1. 基本的な考え方
- **コンテキスト管理:** 文脈（Context Window）は有限です。不要な履歴は`/clear`で削除し、パフォーマンス低下を防ぎます。
- **検証手段の提供:** Claudeが自身で結果を検証できるよう、テストコードや実行コマンド、期待される出力を提示してください。

### 2. 推奨ワークフロー（Explore → Plan → Code）
1. **調査:** `Plan Mode`でファイルを読み込み、構造を理解する。
2. **計画:** 具体的な実装プランを作成させる。
3. **実装:** `Normal Mode`に切り替え、計画に沿ってコードを書かせる。
4. **コミット:** 修正内容を確認し、PRを作成する。

### 3. 環境設定とカスタマイズ
- **CLAUDE.md:** プロジェクト固有のスタイルやコマンドを記述し、セッション開始時に読み込ませます（`/init`で生成可能）。
- **拡張機能:** MCPサーバー、カスタムスキル、サブエージェント（`.claude/agents/`）を活用して機能を拡張します。
- **フック:** ファイル編集後の自動リント実行など、決定論的な処理を`hooks`に設定します。

### 4. コミュニケーションとセッション管理
- **具体性:** `@ファイル名`で対象を指定し、曖昧な指示を避けます。
- **軌道修正:** 誤りに気づいたらすぐに`Esc`で停止し、`/rewind`で戻るか指示を修正します。
- **サブエージェントの活用:** 大規模な調査はメインコンテキストを汚さないよう、サブエージェントに委託します。

### 5. 自動化とスケール
- **ヘッドレスモード:** `claude -p \\"prompt\\"`を使用して、CI/CDやスクリプトから実行します。
- **並列セッション:** Writer（書く役）とReviewer（レビュー役）を分けるなど、複数のセッションを同時に走らせて品質を高めます。
