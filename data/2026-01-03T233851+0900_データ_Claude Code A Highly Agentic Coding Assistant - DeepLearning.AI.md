#データ

1. Claude Codeで効率的に開発する2つのコツ
Claude Codeに指示を出す際、より正確で効率的な結果を得るための重要なポイントが紹介されています。

ファイルを直接指定する
「@ファイル名」と入力することで、Claudeにどのファイルを修正すべきか直接教えることができます。これにより、AIが迷うことなく正確な作業を行えます。
「計画モード（Plan Mode）」を活用する
いきなりコードを書かせるのではなく、まず「何を変えるべきか」という計画案を作成させます。人間がその計画を確認・承認してから作業に入ることで、大きな変更でもミスを防げます。
2. 見た目（UI）を改善する：引用リンクとデザイン
チャットボットの回答の根拠となる「ソース（情報源）」を、単なるテキストではなくクリック可能なリンクにする手順です。

自動承認（Auto-accept）機能
修正箇所が多い場合、一つずつ許可を出すのは大変です。信頼できる作業であれば、一括で修正を許可する設定に切り替えて、スムーズに開発を進められます。
スクリーンショットによる指示
「リンクの色が見にくい」といったデザインの相談は、画面のスクリーンショットをClaudeに貼り付けるだけで伝わります。AIが画像を見て「青色が濃すぎて読みにくいですね」と判断し、最適な色に修正してくれます。
3. 便利機能の追加：新規チャットボタン
ページを更新せずに会話をリセットできる「New Chat（新規チャット）」ボタンを実装します。

作業の「初期化」が大事
新しい機能を作る時は、それまでの会話履歴を一度クリア（/clear）することが推奨されます。これにより、AIが過去の無関係な情報に混乱するのを防げます。
一貫性のあるデザイン
既存のボタンと見た目が揃うように、スタイル（色や配置）も自動で調整させることができます。
4. MCP（Model Context Protocol）による強力な連携
「MCP」という仕組みを使うと、Claude Codeの能力をさらに広げることができます。動画ではPlaywrightというツールとの連携が紹介されています。

ブラウザの自動操作
MCPを使うと、Claudeが自分でブラウザを立ち上げ、作成した画面をチェックし、崩れている箇所を見つけて自ら修正する…といった「全自動のテストと修正」が可能になります。
プロジェクトメモリ（CLAUDE.md）
「サーバーを自動で起動しないでほしい」といった自分専用のルールをメモに残しておくことで、次回以降もその設定を覚えておいてくれます。
5. 裏側（バックエンド）の強化：情報の詳細化
チャットボットがより詳しく答えられるように、検索ツールを強化します。

新しいツールの登録
コースの概要だけでなく、各レッスンの詳細な内容（タイトルや説明）まで取得できるように、裏側のプログラム（Pythonファイル）を書き換えます。
システム指示の更新
プログラムだけでなく、AIに対する「追加データを活用して答えてください」という指示（プロンプト）も自動で更新し、ボットの賢さを底上げします。
まとめ
Claude Codeを使えば、「計画を立てる」→「コードを書く」→「画面を見て修正する」という一連の開発プロセスを、AIとの対話だけで素早く進めることができます。特にスクリーンショットを使ったデザイン修正や、MCPによる外部ツール連携は、開発を劇的に楽にする強力な機能です。
