## Claude Code on the Webの概要
Claude Code on the webは、Claudeアプリから直接コーディングタスクを開始できる開発者向けツールです。

### 主な利点
*   **質問への回答**: コードアーキテクチャや機能実装に関する質問。
*   **バグ修正・定型タスク**: 頻繁な介入が不要なタスク。
*   **並行作業**: 複数のバグ修正を同時に実施。
*   **リモートリポジトリ**: ローカルにチェックアウトしていないコードでの作業。
*   **バックエンド変更**: テスト作成からコード実装まで。

### iOSアプリの利点
*   **外出先での作業**: 移動中やPCから離れていてもタスク開始。
*   **監視**: エージェントの進捗状況を監視し、指示。

### 利用対象者
*   Proユーザー
*   Maxユーザー
*   Team premiumシートユーザー
*   Enterprise premiumシートユーザー

### 利用開始手順
1.  [claude.ai/code](https://claude.ai/code) にアクセス
2.  GitHubアカウントを接続
3.  リポジトリにClaude GitHubアプリをインストール
4.  デフォルト環境を選択
5.  コーディングタスクを送信
6.  変更を確認し、GitHubでプルリクエストを作成

### 動作原理
1.  **リポジトリのクローン**: Anthropic管理の仮想マシンにリポジトリをクローン。
2.  **環境セットアップ**: コードを含むセキュアなクラウド環境を準備。
3.  **ネットワーク設定**: 設定に基づいてインターネットアクセスを構成。
4.  **タスク実行**: コード分析、変更、テスト実行、自己チェック。
5.  **完了通知**: 完了時に通知され、PRを作成可能。
6.  **結果**: 変更はブランチにプッシュされ、PR作成準備完了。

### Webとターミナル間のタスク移動
Webで開始したタスクは、「Open in CLI」ボタンからターミナルに移行し、ローカルで作業を継続できます。

### クラウド環境
*   **デフォルトイメージ**: 主要なツールチェーンや言語エコシステム（Python, Node.js, Ruby, PHP, Java, Go, Rust, C++、PostgreSQL, Redisなど）をプリインストール。
    *   **利用可能なツールの確認**: `check-tools` コマンドを実行。
*   **環境設定**: リポジトリのクローン、ネットワーク設定、タスク実行が行われます。`.env`形式で環境変数を設定可能。
*   **依存関係管理**: `SessionStart`フック (`.claude/settings.json`) を使用して依存関係の自動インストールを設定できます。
    *   スクリプト例: `scripts/install_pkgs.sh` で `npm install` や `pip install` を実行。
    *   **ローカルとリモートの実行区別**: `CLAUDE_CODE_REMOTE` 環境変数で制御。
    *   **環境変数の永続化**: `CLAUDE_ENV_FILE` に書き込むことで永続化。

### ネットワークアクセスとセキュリティ
*   **ネットワークポリシー**: 
    *   **GitHubプロキシ**: すべてのGitHub操作はプロキシ経由。安全な認証と、作業ブランチへのプッシュ制限。
    *   **セキュリティプロキシ**: すべてのアウトバウンドインターネットトラフィックはプロキシを通過。悪意のあるリクエストからの保護、レート制限、コンテンツフィルタリング。
*   **アクセスレベル**: デフォルトでは「制限付き」で、**許可されたドメイン**にのみアクセス可能。
    *   **デフォルトで許可されるドメイン**: Anthropicサービス、バージョン管理、コンテナレジストリ、クラウドプラットフォーム、各種パッケージマネージャー、Linuxディストリビューション、開発ツール、クラウドサービス、コンテンツ配信、スキーマなど多数のドメインが許可されています（ワイルドカードを含む）。
*   **セキュリティベストプラクティス**: 必要最小限のネットワークアクセス、定期的な監査、HTTPSの使用。

### セキュリティと分離
*   **分離された仮想マシン**: 各セッションは分離されたVMで実行。
*   **ネットワークアクセス制御**: デフォルトでネットワークアクセスは制限可能（Anthropic APIとの通信は許可）。
*   **認証情報保護**: 機密性の高い認証情報はサンドボックス内に保存されず、セキュアなプロキシ経由で処理。
*   **安全な分析**: 隔離されたVM内でコードが分析・変更され、PRが作成されます。

### 料金とレート制限
*   Web上のClaude Codeは、アカウント内の他のClaudeおよびClaude Codeの使用とレート制限を共有します。
*   並行して複数のタスクを実行すると、より多くのレート制限を消費します。

### 制限事項
*   **リポジトリ認証**: Webからローカルへのセッション移動には、同じアカウントでの認証が必要。
*   **プラットフォーム制限**: Web上のClaude CodeはGitHubでホストされているコードでのみ機能します。

### ベストプラクティス
1.  **Claude Codeフックの活用**: `SessionStart`フックで環境セットアップや依存関係インストールを自動化。
2.  **要件の文書化**: `CLAUDE.md`ファイルに依存関係とコマンドを明記。`AGENTS.md`をソースすることも可能。
