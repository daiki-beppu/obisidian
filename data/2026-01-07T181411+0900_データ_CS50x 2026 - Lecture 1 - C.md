#CS50 #プログラミング入門 #C言語

CS50 講義要約：配列、メモリ、そしてコンパイルの詳細
1. デバッグの技術とツール
プログラミングにおいて「バグ（間違い）」は避けられないものであり、それを特定し修正するプロセスが不可欠です。
• バグの由来: 「バグ」という言葉は、かつて計算機の回路内に本物の蛾（Moth）が入り込み故障を引き起こしたことに由来します。
• デバッグ手法:
    ◦ printfデバッグ: 変数の値を一時的に画面に表示させ、プログラムの内部状態を確認する最も基本的な手法です,。
    ◦ debug50: CS50が提供するツールで、プログラムを一行ずつ実行（ステップ実行）し、メモリ内の変数の変化を視覚的に観察できます,。
    ◦ ラバーダック・デバッグ:  inanimate（無生物）の対象（アヒルなど）にコードの論理を説明することで、自分自身の論理的ミスに気づく手法です。
2. コンパイルの4つの段階
ソースコードがマシンコード（0と1）に変換されるまでには、実際には4つの連続したプロセスが行われています。
1. プリプロセッシング (Pre-processing): #include で指定されたヘッダーファイルの内容をソースコードにコピー・展開します,。
2. コンパイル (Compiling): C言語のコードを、CPUが理解できる低レベルなアセンブリ言語に変換します,。
3. アセンブリ (Assembly): アセンブリ言語をバイナリのマシンコードに変換します,。
4. リンキング (Linking): 自分のコード、CS50ライブラリ、標準ライブラリなどの複数のバイナリファイルを1つの実行ファイルに結合します,。
3. メモリとデータ型
コンピューターのメモリ（RAM）は、データを保存するための「バイトの格子（グリッド）」として捉えることができます,。
• データ型ごとのサイズ:
    ◦ bool: 1バイト。
    ◦ int / float: 4バイト,。
    ◦ long / double: 8バイト。
    ◦ char: 1バイト。
• ガベージバリュー (Garbage Value): 変数を初期化せずに宣言すると、以前そのメモリ場所を使っていたプログラムの残骸（ゴミの値）が入っていることがあります,。
4. 配列 (Arrays)
配列は、同じ型のデータをメモリ上に「連続して（contiguous）」配置した構造です,。
• 利点: 複数の関連する値（例：試験のスコア）を1つの変数名で管理でき、ループ処理との相性が非常に良いです,。
• インデックス: C言語の配列は常に 0 から数え始めます。
• C言語の制約: C言語の配列は自分自身のサイズを記憶していないため、関数に配列を渡す際は、その長さ（length）も一緒に渡す必要があります,。
5. 文字列 (Strings) の正体
C言語において、文字列は単なる文字（char）の配列に過ぎません,。
• ヌル終端文字 (\0): 文字列の終わりをコンピューターに示すため、メモリの最後に全ビットが0の特別なバイト（\0）が自動的に挿入されます,。
• メモリ上のサイズ: 例えば "HI!" という3文字の文字列を保存するには、ヌル文字を含めて4バイトのメモリが必要です。
• ライブラリ: string.h の strlen 関数を使えば文字列の長さを取得でき、ctype.h を使えば文字の大文字・小文字変換などが容易に行えます,。
6. コマンドライン引数と終了ステータス
プログラム実行時に、ターミナルから直接データを渡す仕組みです。
• argc と argv: main 関数は、引数の個数（argc）と、引数の文字列配列（argv）を受け取ることができます,。なお、argv には常に実行したプログラム名が入ります。
• 終了ステータス: main 関数が返す整数値です。通常、0は成功を意味し、それ以外（1や2など）はエラーが発生したことをOSに伝えます,。
7. 暗号化 (Cryptography)
情報をスクランブルして秘匿する技術です。
• 平文 (Plaintext) と 暗号文 (Ciphertext): 人間が読めるメッセージを、アルゴリズムと「鍵（Key）」を用いて読めない形に変換します。
• シーザー暗号: 文字をアルファベット順に一定数（鍵）だけずらす古典的な手法です（例：鍵が1なら A→B）,。
