## Claude Code Chrome連携

Claude Codeは、Chromeブラウザ拡張機能と連携し、ターミナルから直接ブラウザの自動化を可能にします。これにより、ターミナルで開発し、ブラウザでテスト・デバッグするという一連の作業をコンテキストを切り替えることなく行えます。

### 主な機能
*   **ライブデバッグ**: コンソールエラーやDOMの状態を直接読み取り、コードを修正
*   **デザイン検証**: FigmaモックからUIを構築し、ブラウザで一致を検証
*   **Webアプリテスト**: フォーム検証、視覚的回帰テスト、ユーザーフローの検証
*   **認証済みWebアプリ**: Google Docs, Gmail, Notionなど、ログイン済みのアプリと連携
*   **データ抽出**: Webページから構造化された情報を取得
*   **タスク自動化**: データ入力、フォーム入力、複数サイトのワークフローなどを自動化
*   **セッション記録**: ブラウザ操作をGIFとして記録

### 前提条件
*   Google Chromeブラウザ
*   Claude in Chrome拡張機能 (v1.0.36以上)
*   Claude Code CLI (v2.0.73以上)
*   有料Claudeプラン (Pro, Team, Enterprise)

### 仕組み
Claude Codeは、Claude in Chromeブラウザ拡張機能を通じてChromeと通信します。拡張機能はChromeのNative Messaging APIを使用し、Claude Codeからのコマンドをブラウザで実行します。これにより、ターミナル作業中にブラウザタブの制御、ページコンテンツの読み取り、アクションの実行が可能になります。

*   ログインページなどでブロッカーに遭遇すると一時停止し、手動での対応を求めます。
*   既存のタブではなく、新しいタブを開いてブラウザタスクを実行します。
*   ブラウザのログイン状態を共有するため、再認証なしでログイン済みサイトにアクセスできます。
*   ヘッドレスモードはサポートされておらず、ブラウザウィンドウが常に表示されます。

### セットアップ手順
1.  **Claude Codeを更新**: `claude update` を実行 (ネイティブインストーラー利用の場合は自動)
2.  **Chrome連携を有効にして起動**: `claude --chrome` コマンドでClaude Codeを起動
3.  **接続を確認**: `/chrome` コマンドで接続状態と設定を管理

### 使用例
*   ローカルWebアプリケーションのテスト
*   コンソールログを使ったデバッグ
*   フォーム入力の自動化
*   Google Docsでのコンテンツ作成
*   Webページからのデータ抽出
*   複数サイトにまたがるワークフローの実行
*   デモGIFの記録

### ベストプラクティス
*   モーダルダイアログ (JavaScript alerts, confirms, prompts) はフローを中断する可能性があるため注意
*   常に新しいタブを使用
*   コンソール出力はフィルタリングして目的のパターンを探す

### トラブルシューティング
*   **拡張機能が検出されない**: 拡張機能/CLIのバージョン確認、Chrome起動確認、`/chrome`で「Reconnect extension」、両方の再起動
*   **ブラウザが応答しない**: モーダルダイアログの確認、新しいタブの作成、拡張機能の無効化/再有効化
*   **初回セットアップ**: 権限エラーの場合、Chromeの再起動が必要な場合あり

### デフォルトで有効化
`/chrome` で「Enabled by default」を選択すると、`--chrome` フラグなしで常にChrome連携を有効にできます。ただし、コンテキスト使用量が増加する可能性があるため、必要に応じて設定してください。
